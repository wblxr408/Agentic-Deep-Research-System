# 后端开发

### 异步桩必须和被测实现的 await 语义一致

- 问题：服务层测试里，如果真实实现对依赖使用了 `await`，测试桩也必须是协程桩；如果误用同步 lambda 或普通对象，测试可能在弱覆盖路径下形成假阳性。
- 这次对应场景：`skill` 服务层版本测试里，`get_skill_meta_by_id`、`get_skill_meta_by_slug`、`get_db_pool`、`get_skill_content` 这类依赖在实现中都是异步调用，测试里必须使用 `async def` 或 `AsyncMock`。
- 处理原则：
  - 先看实现签名和调用点，确认依赖是同步还是异步。
  - 只要实现里有 `await dep(...)`，测试替身就必须可 await。
  - 对数据库连接、外部服务、缓存客户端这类边界依赖，优先使用异步假对象或 `AsyncMock`。
- 价值：避免测试“看起来通过”，但其实没有覆盖真实 await 路径。

### 文档类问题单独记，不混入 Agent 面试笔记

- 原则：如果问题主要是后端工程、测试习惯、构建链路、数据库结构、缓存策略这类内容，不强行塞进 Agent 面试材料。
- 做法：单独沉淀到这份笔记，后续继续累计测试陷阱、接口边界、数据迁移和缓存一致性问题。
